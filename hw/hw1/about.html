Man-In-Middle Attack: what is it?

Man in the middle attack involves a hacker intercepting packets that are being transmitted from one server
The hacker starts this process by using different software to "listen" on the packet transmission. From there
They choose what packets seem the most valuable to them and intercept those packets. As a result, the spoofed
packet is sent to the server where it will either be accepted or denied by the recieving server. Usally the
server will be able to identify if the packet has been tampered with, thus denying access. However, if the hacker is
successful, then they will be able to send false information or use the phising technique to gain more
data from the user.
